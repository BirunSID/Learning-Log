# Knowledge Vault (The Engine)

## Purpose
The Knowledge Vault is a **synthesized repository** of core concepts, theories, and mental models. Unlike the `Logs` folder (which tracks *time*) or the `Labs` folder (which stores *files*), this vault is dedicated to **Knowledge**. 

Every file in here is a "Permanent Note"—meaning it has been processed, rewritten in my own words, and structured for long-term storing.

---

## Directory Structure
This vault is organized by domain to ensure scalability as the "Human-OS"(self-development) evolves:

```text
Knowledge_Vault/
├── 📂 1_MS_Excel/                     # Spreadsheet, tips and tricks and how it all works.
├── 📂 2_Math/                         # Linear Algebra, Calculus, and Statistics for ML/DS.
├── 📂 3_Learning_German(Deutsch)/     # Vocabularies and resources and consistent practice.
└── 📂 Assets/                         # Diagrams, math notations, and visual aids for notes.
```

---

## The "Vault" Rules
To maintain professionalism and prevent clutter, this directory follows strict architectural rules:

1. **Synthesized Only:** No raw "copy-pastes" from lectures. Notes must be processed and simplified.
2. **No Chronological Logs:** Daily progress belongs in `/Logs`. This space is for permanent theory only.
3. **Atomic Documentation:** Each file should focus on a specific concept (e.g., `Matrix-Multiplication.md` rather than one giant `Math.md`).
4. **The "Grandmother" Rule:** Write the notes as if I'm are explaining the concept to someone else. If I can't explain it simply, I don't understand it yet.
