## Directory Structure
This folder is organized by domain, ensuring that each technical track has its own dedicated workspace for implementation:

```text
Projects_and_Labs/
├── 📂 1_CS50x/          # Source code (.c, .py, .sql) and local documentation of problems sets.
├── 📂 2_MS_Excel/       # Functional .xlsx files, and test files. very messy.
```

## 📜 The "Lab" Rules
To ensure this repository remains a professional portfolio, the following standards are maintained:

1. **Functional Deliverables only:** Every file in this directory should be "executable" or "viewable" as a completed piece of work.
2. **Standardized Documentation:** Major projects (like CS50x Psets) must include an internal `README.md` explaining the problem solved and the logic used.
3. **No Theory Clutter:** Synthesized notes and definitions belong in the `/Knowledge_Vault`. Keep this space for implementation only.
4. **Version Control:** All code should be clean, commented, and refactored for readability.
